# הדרכת סימניות עם כפתור דמה

## סקירה כללית
מערכת ההדרכה המשופרת כוללת כפתור דמה שמוצג באופן מלאכותי כדי להדגים איך הוספת סימניות עובדת, גם כשהכפתור האמיתי לא נראה.

## הבעיה שנפתרה
- כפתורי הוספת סימניה ב-ChatGPT מוגדרים עם `opacity-0 group-hover:opacity-100`
- הם נראים רק כשמרחפים על ההודעה
- ההדרכה לא יכלה להצביע עליהם כי הם לא נראו

## הפתרון החדש

### 1. יצירת כפתור דמה
```typescript
function createDemoBookmarkButton(): HTMLElement
```
- יוצר כפתור זהה לזה האמיתי
- מוצב ליד הודעה קיימת או במיקום נופרקט
- נראה בבירור עם אפקט זוהר (`box-shadow` ו-`animation`)

### 2. מיקום חכם
הכפתור הדמה מוצב לפי עדיפויות:
1. ליד ההודעה האחרונה של העוזר
2. ליד ההודעה האחרונה של המשתמש  
3. ליד כל הודעה קיימת
4. במיקום נתפקט (fallback)

### 3. ניקוי אוטומטי
- הכפתור הדמה נמחק בסוף ההדרכה
- גם ה-CSS styles נמחקים
- לא משפיע על הפונקציונליות הרגילה

### 4. אפקטים ויזואליים
- זוהר כחול (`box-shadow`)
- אנימציית דופק (`@keyframes tutorial-pulse`)
- גודל וצבעים זהים לכפתור האמיתי

## הטכנולוגיות בשימוש

### CSS
```css
@keyframes tutorial-pulse {
  0%, 100% { box-shadow: 0 0 0 2px rgba(59, 130, 246, 0.5); }
  50% { box-shadow: 0 0 0 4px rgba(59, 130, 246, 0.3); }
}
```

### DOM Manipulation
- `createElement` ליצירת הכפתור
- `appendChild` להוספה לעמוד
- `querySelector` למציאת מיקום מתאים

### Event Handling
- ניקוי הכפתור בכל אירועי סגירת ההדרכה
- הבטחה שלא נשארים רכיבים מיותרים

## יתרונות
1. ✅ ההדרכה תמיד עובדת, גם אם הכפתור האמיתי לא נראה
2. ✅ המשתמש רואה בדיוק איך הכפתור נראה
3. ✅ לא משפיע על הפונקציונליות הרגילה
4. ✅ ניקוי אוטומטי מונע זיהום של העמוד
5. ✅ אפקטים ויזואליים מושכים תשומת לב

## אינטגרציה עם המערכת הקיימת
- נשמר תואם למערכת ההדרכה הקיימת
- משתמש באותם סלקטורים וקלאסים
- ניתן לזהות בקלות עם `[data-demo-bookmark="true"]`
